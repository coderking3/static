# Prompt 指南 · 阶段一：项目规划

> 还没写代码。方向未定，需要 AI 帮你想清楚做什么、怎么做、用什么技术。

---

## 场景一：技术选型

> 几个方案里选一个，需要对比分析后给出推荐。

**适用**：框架选型、库选型、架构模式选择、数据库选择。

```markdown
# 调研：[要解决的问题]

## 背景

[为什么要选型，当前面临什么问题]

## 候选方案

- 方案A：[已知选项]
- 方案B：[已知选项]
- 也许还有其他方案？

## 评估维度

[性能 / 成本 / 维护性 / 团队学习曲线 / 生态 ...]

## 请你做

1. 列出所有可行方案（包括我没提到的）
2. 按上述维度逐个对比
3. 给出推荐方案和理由
4. 列出推荐方案的落地步骤

## 约束

- 结合我们项目实际情况（技术栈：xxx，团队规模：xxx）
- 不要只列优缺点，要给出明确推荐
- 如果推荐方案有风险，标出来
```

---

## 场景二：方案设计

> 有模糊需求，需要 AI 先分析、再访谈你、最后出完整方案。

**适用**：新项目启动、复杂需求拆解、不确定怎么做。

```markdown
# 需求：[模糊标题]

## 背景

[业务背景、为什么要做]

## 初步想法

[我目前想到的方向，可能不完整]

## 请你做

1. 先分析需求目标和关键决策点
2. 对我进行访谈，逐个问我不确定的点
3. 根据我的回答，输出完整技术方案
4. 方案需包含：技术选型、模块拆分、接口设计、风险点

## 约束

- 访谈时一次只问 1-2 个问题，等我回答后再继续
- 不要假设我懂所有技术细节，用我能理解的方式问
- 方案要可落地，不要写空话
```

## 场景三：需求拆解

> PRD 拿到了，要拆成可执行的技术任务。

**适用**：需求评审后、Sprint 规划、任务分配。

```markdown
# 需求拆解：[PRD 标题]

## PRD

[链接 或 贴关键内容]

## 项目背景

- 技术栈：[xxx]
- 相关模块：[xxx]

## 请你做

1. 通读 PRD，提炼核心功能点
2. 拆成可独立执行的技术任务（粒度：0.5-2 天）
3. 标注任务间依赖关系
4. 标注风险点和不确定项

## 约束

- 任务粒度不要太粗，能独立分配和验收
- 不要遗漏异常流程和边界情况
```

## 场景四：学习理解

> 读不懂一段代码，或想了解一个概念 / 库。

**适用**：接手陌生代码、学习新技术、理解架构设计。

```markdown
# 理解：[想理解什么]

## 我的困惑

[哪里看不懂 / 什么概念不清楚]

## 代码 / 资料

[贴代码 或 文件路径 或 链接]

## 我的背景

[我对这个领域的了解程度，如：我懂 React 但没用过 Redux]

## 请你做

- 用我能理解的方式解释
- 结合代码具体行号说明
- 如果有更简单的实现方式，指出来
```

---

## 本阶段原则

- **不要急着要代码**：这个阶段的目标是"想清楚"，不是"写出来"
- **让 AI 问你**：信息不全时，让 AI 访谈你，而不是它自己猜
- **给业务背景**：AI 不懂你的业务，"为什么要做"比"做什么"更重要
- **要推荐不要罗列**：选型时明确要求 AI 给推荐，不要"各有优劣"的废话
